## Spotify Discord Hosts Fixer

> **Неофициальный** инструмент с открытым кодом. Не связан со Spotify, Discord или GeoHide.
> Используйте на свой страх и риск.

Десктоп-программа (Windows / macOS / Linux), которая помогает восстановить отображение
«сейчас слушает Spotify» в Discord в условиях, когда домены Spotify недоступны или
замедлены на вашем интернет-канале. Программа добавляет в системный файл `hosts`
блок записей, перенаправляющий домены Spotify на прокси-узлы GeoHide.

## Что именно делает программа

Получает IP прокси-узлов через DNS-резолв `geohide.ru` и проверяет их доступность по TCP `:443`.
По нажатию «Обновить и применить» запрашивает права администратора (UAC), делает
резервную копию `hosts` (`hosts.backup.<дата_время>`) и записывает блок между маркерами
`#spotify-discord-hosts` … `#end-spotify-discord-hosts`. В hosts записывается **один лучший
узел** (с наименьшей задержкой) — система всё равно использует только первую запись для домена.
После записи программа проверяет, что перенаправление реально работает (резолвит домен через
системный резолвер), и далее периодически перепроверяет активный узел, предупреждая, если он отвалился.
По нажатию «Сбросить hosts» удаляет этот блок и восстанавливает стандартную маршрутизацию.
Сбрасывает кэш DNS после изменения.

## ⚠️ Важно про безопасность

По умолчанию программа перенаправляет домены Spotify — включая домены авторизации
(`accounts.spotify.com`, `login5.spotify.com`) — на серверы GeoHide. Это значит, что
весь соответствующий трафик будет идти через сторонний сервис, которому вы должны
доверять. Программа не устанавливает корневые сертификаты и не перехватывает
зашифрованный трафик сама по себе, но конечная безопасность зависит от того, что
делают серверы GeoHide. Решение об использовании — за вами.

**Минимальный режим** (переключатель в настройках) исключает домены авторизации:
перенаправляются только домены, нужные для презенса/синхронизации, а логин в Spotify
идёт напрямую, минуя прокси. Рекомендуется, если вам важно не пропускать поток
авторизации через сторонний сервис.

Изменения вносятся только после явного запроса прав администратора и
окна-согласия при первом применении; всегда создаётся резервная копия, и блок полностью обратим.

## Установка (готовый .exe)

Скачайте установщик или portable-версию со страницы [Releases](https://github.com/swd3k/spotify-discord-hosts-fixer/releases) и запустите.

**Проверка целостности.** К каждому релизу прилагается файл `SHA256SUMS.txt` с контрольными
суммами. Сравнить можно командой PowerShell:

```powershell
Get-FileHash .\Spotify*.exe -Algorithm SHA256
```

**Предупреждение SmartScreen.** Исполняемый файл не подписан сертификатом (подпись стоит денег),
поэтому Windows SmartScreen может показать предупреждение «Защитник Windows SmartScreen предотвратил
запуск». Если вы доверяете источнику (сверили чексумму с релизом на GitHub), нажмите
«Подробнее» → «Выполнить в любом случае».

**Автообновление.** Установленная версия (через установщик NSIS) сама проверяет обновления
на GitHub Releases и предлагает их установить. Portable-версия не обновляется автоматически —
скачивайте новую вручную.

## Возможности

- Один лучший прокси-узел (по задержке) на все домены + проверка после применения
- Периодическая перепроверка активного узла с предупреждением, если он недоступен
- Минимальный режим — без перенаправления доменов авторизации
- Предпросмотр блока hosts и просмотр реально установленного блока
- Автозапуск вместе с Windows (свёрнуто в трей)
- Запоминание размера и позиции окна, светлая/тёмная тема

## Сборка из исходников

Требуется Node.js 20+.
```bash
npm install
npm run dist      # соберёт установщик и portable .exe в папку release/
```

## Для разработки

```bash
npm install
npm run dev:renderer   # дев-сервер интерфейса (Vite)
npm run typecheck      # проверка типов
npm run lint           # ESLint
npm test               # тесты логики hosts (vitest)
```

CI собирает проект на каждый push в `main` и pull request. Релиз создаётся по тегу `v*`;
версия приложения подставляется из тега, поэтому имя `.exe` и тег не могут разойтись.

## Обрабатываемые домены

Полный режим:
```
api.spotify.com
login5.spotify.com *
encore.scdn.co
gew1-spclient.spotify.com
spclient.wg.spotify.com
api-partner.spotify.com
aet.spotify.com
www.spotify.com *
accounts.spotify.com *
open.spotify.com
accounts.scdn.co *
gew1-dealer.spotify.com
open-exp.spotifycdn.com
www-growth.scdn.co
```
* — домены авторизации, исключаются в минимальном режиме.

## Отказ от ответственности

Инструмент предназначен для восстановления штатной работы легально
используемых сервисов. Изменение системного hosts — на ваш страх и риск;
перед каждым изменением создаётся резервная копия, доступен откат.
