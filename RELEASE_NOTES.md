# v1.5.0

## Новое
- **Горизонтальный интерфейс.** Окно стало широким, контент разложен в две колонки: слева — статус, управление и настройки, справа — список прокси-узлов, предпросмотр hosts и журнал. Больше не нужно прокручивать длинную колонку.

## Изменения
- **Минимальный режим удалён.** Перенаправляется единый полный набор доменов Spotify; переключатель из настроек убран. Окно-согласие перед первым применением по-прежнему описывает, что именно делает программа.

## Установка
Скачайте `Setup`-установщик или portable-версию ниже и запустите. Установленные v1.4.2+ предложат обновиться сами.

⚠️ Неофициальный проект, не связан со Spotify / Discord / GeoHide. Трафик перенаправляемых доменов Spotify (включая авторизацию) идёт через сторонний сервис GeoHide — используйте на свой риск.

# v1.4.2

## Исправления
- **Починено автообновление.** Updater не мог скачать новую версию: имя файла в `latest.yml` не совпадало с именем файла в релизе (GitHub заменял пробелы в имени на точки, а updater искал имя с дефисами и получал 404). Файлы релиза теперь именуются без пробелов.
- **Обновление предлагается явно.** Вместо тихого системного уведомления — диалог «Установить и перезапустить / Позже». Если выбрать «Позже», обновление установится при выходе из программы.
- Проверка обновлений выполняется не только при запуске, но и каждые 4 часа — приложение часто неделями живёт в трее.
- Ошибки обновления теперь пишутся в `updater.log` (папка данных приложения) — раньше они молча проглатывались.

## Установка
Скачайте `Setup`-установщик или portable-версию ниже и запустите.

Если у вас установлена v1.4.0 или v1.4.1 — перезапустите программу: она сама скачает это обновление и установит его при выходе (явный диалог появится начиная с этой версии).

⚠️ Неофициальный проект, не связан со Spotify / Discord / GeoHide. Трафик перенаправляемых доменов Spotify идёт через сторонний сервис GeoHide — используйте на свой риск. Включите «Минимальный режим», чтобы не пропускать авторизацию через прокси.

# v1.4.0

## Новое
- **Минимальный режим** (переключатель в настройках): домены авторизации (`accounts.spotify.com`, `login5.spotify.com`) не перенаправляются — логин в Spotify идёт напрямую, минуя прокси.
- **Один лучший узел.** В hosts записывается только узел с наименьшей задержкой, а не все подряд — система всё равно использует лишь первую запись для домена.
- **Проверка после применения.** Программа резолвит домен через системный резолвер и подтверждает, что перенаправление реально работает.
- **Мониторинг узла.** Активный узел периодически перепроверяется; если он перестал отвечать — предупреждение в логе и уведомление.
- **Окно-согласие** перед первым применением: понятное описание того, что именно изменит программа, с учётом выбранного режима.
- **Запуск вместе с Windows** (свёрнуто в трей) — переключатель в настройках.
- **Текущий блок hosts** теперь виден прямо в приложении — рядом с предпросмотром показывается реально установленный блок.
- **Автообновление.** Установленная версия сама проверяет новые релизы на GitHub и предлагает обновиться (portable-версия — нет, скачивайте вручную).
- Запоминание размера и позиции окна между запусками.

## Сборка и безопасность
- CI теперь проверяет сборку (типы, линт, тесты) на каждый push и pull request, а не только на тег.
- Версия приложения берётся из git-тега — имя `.exe` и тег не могут разойтись.
- К релизу прикладывается `SHA256SUMS.txt` с контрольными суммами; в README — как их проверить и как обойти предупреждение SmartScreen.
- Логика формирования и разбора блока hosts покрыта тестами (vitest); подключены ESLint и Prettier.

# v1.3.0

## Новое
- **Работа в трее.** Значок в области уведомлений с меню («Открыть», «Скрыть в трей», «Выход»); клик по значку показывает/прячет окно. Закрытие окна сворачивает приложение в трей, а не завершает его.
- **Иконка приложения.** Добавлена иконка (`.ico` для `.exe`, отдельная иконка для трея и окна).
- **Полупрозрачный фон окна** (acrylic на Windows 11, vibrancy на macOS). На Windows 10 / Linux эффект плавно деградирует до полупрозрачной заливки.
- **Ссылка на проект** в шапке и в блоке помощи — открывается в системном браузере.
- Запрет повторного запуска: второй экземпляр разворачивает уже открытое окно.

## Исправления
- Исправлена невидимость иконки переключения темы при наведении.
- Устранён корень проблемы — в разметке использовались несуществующие в Tailwind v4 оттенки (`neutral-150/250/450/750/850`, `rose-450`), из-за чего «пропадали» рамки, фоны и подсветка. Оттенки объявлены явно.
- Убрано лишнее системное меню Electron (File / Edit / View / Window / Help).
- Устранено мелькание белым фоном при запуске.
- Обработчик внешних ссылок ограничен схемами `http/https`.
- Версия в шапке приложения теперь берётся из package.json и не расходится с релизом.
- Кнопка «Обновить список» больше не прокручивает страницу вниз — автоскролл работает только внутри блока логов.
